Important Note about Print Sources:

So, typically, a print source would be an actual book, encyclopedia, magazine article, etc; that you could physically read, but since NTI is crazy, I will let you use online sources that are actually published. For example, the New York Times is online, and it is also published in print. I will let you use things like that. You just have to be careful that you know it is published in print. For example, the Huffington Post is only on-line, so you could not count that as a print source . Hopefully, that clears up any confusion. Two of your five sources must be print.

    Audio and Video Elements Overview

    https://developer.mozilla.org/en-US/docs/Learn/HTML/Multimedia_and_embedding/ Video_and_audio_content

    Both of these elements allow for multimedia content to be embedded directly on a webpage without external plugins such as Flash

    As an alternative to these elements, many Online Video Providers (OVPs) such as YouTube, Dailymotion, Vimeo, and Soundcloud offer the ability to host and consume multimedia content which can save bandwidth

    Note - There are numerous audio/video file format conversion tools available online if a file needs to be changed from one type to another

    Page 3 of 12

    Audio

    https://caniuse.com/#feat=audio

    Supported on all major browsers

    Best format for browser compatibility is .mp3 as this is supported by all major browsers, although other formats such as .ogg and .wav can be used with the Audio element

    Other media properties:

    currentSrc, currentTime, paused, playbackRate, buffered, duration, played, seekable, ended, autoplay, loop, controls, volume & muted

    Page 4 of 12

    Audio

    Sample code:

    <audio controls preload="metadata"> <source src="song.mp3" type="audio/mp3"> <p>Your browser doesn't support HTML5 audio. Here is a <a href="song.mp3">link to the audio</a> instead.</p> </audio>

    Adding controls allows playback control to start/stop the audio and control the volume

    The preload= can be set to "none", "metadata" (basic info) or “auto" (full audio - slow!)

    The type= “audio/mp3” or other is the MIME definition of the filetype

    The paragraph below the open audio tag is a fallback to a direct link to download the audio file for someone who may be using an unsupported browser

    Page 5 of 12

    Video

    https://caniuse.com/#feat=video

    Supported on all major browsers

    Best format for browser compatibility is .mp4 as this is supported by all major browsers, although other formats such as .webm, and .ogg can be used with the Video element

    The video element offers many media properties that can be customized such as:

    currentSrc, currentTime, paused, playbackRate, buffered, duration, played, seekable, ended, autoplay, loop, controls, volume & muted

    Note that this still image displayed in the video player is the “poster”, see the next slide for an code example of this attribute

    Page 6 of 12

    Video

    Sample code:

    <video controls width="400" height="400" poster="poster.png" preload="metadata"> <source src="song.mp4" type="video/mp4"> <p>Your browser doesn't support HTML5 video. Here is a <a href="song.mp4">link to the video</a> instead.</p> </video>

    Adding controls allows playback control to start/stop the video and control the volume

    The height and width can be set directly within the open video tag

    Autoplay, loop, and muted are controls that are optional and are included in the video element; preload= can be set to "none", "metadata" (basic info) or “auto" (full video - slow!)

    The type= “video/mp4” or other is the MIME definition of the filetype

    The poster is the image that will show before the video is played

    The paragraph below the open audio tag is a fallback to a direct link to the video file for someone who may be using an unsupported browser

    Making them responsive

    Give both the audio and video elements a max-width of 100% and the have the video player height set to auto in the style sheet to ensure they don’t fall outside of the their containing grid-item:

    audio { max-width: 100%; } video { max-width: 100%; height:auto; }

    .video, .audio { margin:0; display: grid; /*This needs to be set or it won't center*/ justify-content:space-around; }
